# network-mop
Method of Procedure (MOP) for enabling OSPF dynamic routing on multi-router topologies — demonstrating change management practices aligned with network operations workflows.

Network Change MOP — OSPF Configuration
Overview
This repository contains a Method of Procedure (MOP) document for enabling OSPF dynamic routing between two routers in a multi-network topology. It demonstrates understanding of formal change management processes including pre-checks, implementation steps, post-validation, and rollback procedures.

Skills Demonstrated
Network change management & CR/MOP workflows
OSPF configuration on Cisco routers
Risk assessment and impact analysis
Network validation and troubleshooting
Rollback planning and execution
Tools Used
Cisco Packet Tracer
OSPF routing protocol
Static & dynamic routing concepts
Why This Matters
This MOP follows industry-standard change management practices aligned with how network operations teams handle network changes — from change request approval through implementation, validation, and rollback.
